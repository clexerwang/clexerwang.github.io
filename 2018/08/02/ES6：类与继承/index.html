    ES6：类与继承

    Clexer
    2018-08-02

    @(JavaScript知识点)

    ##ES6：类与继承
    在Es6中新增了class关键字，用来表示类的声明，extends用来表示类的继承。

    例如在React中就有类似的继承：

    class App extends React.Component{
    constructor(){
        super();
        this.state = {
            time: new Date().toLocalTimeString()
        }
    }
    sayTime = ()=>{
        console.log(this.state.time);
    }


    render(){
        return（
            <div>
                <button onClick = {this.sayTime}>Start</button>
            </div>
            ）
    }
}

    ES6中的类和继承是ES5中继承的语法糖，但是他们之间仍有一些不同：

    类声明不能被提升;
    类中的所有方法都是不可枚举的;
    使用关键字new以外的方式调用类的构造函数会报错;
    不可以在类中修改类名；

    知道了以上区别，就可以写出等价代码：

    let App = (function(){
    'use strict'

    const App = function(...args){
        // 类的构造函数只可以通过new调用
        if(new.target === undefined){
            throw new Error('')；
        }
        //借用构造函数
        React.Component.apply(this,args);

        this.name = {
            time: new Date().toLocalTimeString()
        }
    }

    App.prototype = Object.create(React.Component.prototype,{
        constructor:{
            value:App,
            configurable:true,
            enumerable:true,
            writable:true
        },
        syaName:{
            value:function(){},
            enumerable:false, //不可枚举
            configurable:true,
            writable:true
        }    
    })

    // 为了能够使用父类的静态方法
    // 本来App.__proto__指向的是Function.prototype,
    // 这里App.__proto__指向了父类的构造函数，
    // 那么当执行App.sayAge时，就会沿着__proto__去原型对象中寻找对应的方法，这样就能够使用父类的静态方法了
    App.__proto__ = React.Component

    return App;
})()

    静态成员

    在Es6中，类的静态方法通过关键字static来设置。但是static不能用在构造函数前。同时对于父类的静态方法，子类也可以使用。

    继承内建对象

    ES5中无法通过去继承内建对象，比如：Array，Date等。
    是因为在ES5中，首先是使用子类的构造函数生成对象，再使用父类去修饰这个对象。而在ES6中是先使用父类的构造函数生成对象，再使用子类的构造函数去修改它，所以可以访问到父类的内建功能。

    转义后，myDate的构造函数是由一个立即执行函数返回的,父类构造函数_Date由_wrapNativeSuper处理后返回，所以先来看一下这个函数所做的事情：

    _wrapNativeSuper

    可以看到所做的就是让构造函数Wrapper使用原型式继承的方式继承了Date。然后让Warpper.__proto__ = Date 这一步是用来继承Date的静态方法。最后将Warpper函数返回。

    现在返回之前的部分，接着看那个立即执行函数的内部，执行了_inherits(myDate, _Date);，继续看_inherits：

    _inherit


    subClass就是构造函数myDate,superClass就是Warpper。所以，它又执行了一次原型式继承，同样让myDate.__proto__ = Warpper。

    在执行完_inherit以后就得到了处理后的myDate构造函数，现在就看一下在new一个myDate实例时发生了什么。

_classCallCheck(this, myDate);

    _getPrototypeOf(myDate)

    _construct(Date, arguments, _getPrototypeOf(this).constructor); //this为传入的myDate实例

    所以内部所做的就是：emmm,用父类构造函数生成了实例，然后修改了实例的__proto__属性并返回。
    最后一步，执行_possibleConstructorReturn,参数是一个myDate实例，一个Date实例。

    总结：
    整个过程所做的就是:原型式继承，修改子类构造函数的__proto__从而继承父类的静态方法，在子类构造函数中先生成父类实例再修改实例的__proto__属性，最后再为实例添加方法。
