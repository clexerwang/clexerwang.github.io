ES6：类与继承

Clexer
2018-08-02

在Es6中新增了class关键字，用来表示类的声明，extends用来表示类的继承。

例如在React中就有类似的继承：

class App extends React.Component{
    constructor(){
        super();
        this.state = {
            time: new Date().toLocalTimeString()
        }
    }
    sayTime = ()=>{
        console.log(this.state.time);
    }


    render(){
        return（
            <div>
                <button onClick = {this.sayTime}>Start</button>
            </div>
            ）
    }
}

ES6中的类和继承是ES5中继承的语法糖，但是他们之间仍有一些不同：

类声明不能被提升;
类中的所有方法都是不可枚举的;
使用关键字new以外的方式调用类的构造函数会报错;
不可以在类中修改类名；

知道了以上区别，就可以写出等价代码：

let App = (function(){
    'use strict'

    const App = function(...args){
        // 类的构造函数只可以通过new调用
        if(new.target === undefined){
            throw new Error('')；
        }
        //借用构造函数
        React.Component.apply(this,args);

        this.name = {
            time: new Date().toLocalTimeString()
        }
    }

    App.prototype = Object.create(React.Component.prototype,{
        constructor:{
            value:App,
            configurable:true,
            enumerable:true,
            writable:true
        },
        syaName:{
            value:function(){},
            enumerable:false, //不可枚举
            configurable:true,
            writable:true
        }    
    })

    // 为了能够使用父类的静态方法
    // 本来App.__proto__指向的是Function.prototype,
    // 这里App.__proto__指向了父类的构造函数，
    // 那么当执行App.sayAge时，就会沿着__proto__去原型对象中寻找对应的方法，这样就能够使用父类的静态方法了
    App.__proto__ = React.Component

    return App;
})()

静态成员

在Es6中，类的静态方法通过关键字static来设置。但是static不能用在构造函数前。同时对于父类的静态方法，子类也可以使用。

继承内建对象

ES5中无法通过去继承内建对象，比如：Array，Date等。
是因为在ES5中，首先是使用子类的构造函数生成对象，再使用父类去修饰这个对象。而在ES6中是先使用父类的构造函数生成对象，再使用子类的构造函数去修改它，所以可以访问到父类的内建功能。
