浏览器缓存

Clexer
2018-12-12

一、缓存分类

缓存主要分为两种类型：
强缓存：对资源的请求直接从本地的客户端缓存中读取，不与服务器发生交互。

协商缓存：对资源的请求需要发送至服务器，由服务器决定是否使用本地缓存，协商缓存不能减少请求的数量，只会减少响应的体积。

二、强缓存

强制缓存的含义是，当客户端请求后，会先访问缓存数据库看缓存是否存在。如果存在则直接返回，状态码为200；不存在则请求服务器，响应后再写入缓存数据库。

与强缓存有关的头部字段是：Cache-control和Expires

2-1、Expires

Expires是HTTP1.0的字段，它使用一个绝对时间来表示缓存的有效期。在响应头中设置该字段后，浏览器就可以通过时间对比来判断是否要发送请求。

2-2、Cache-control

Cache-control是HTTP1.1中的字段，使用一个相对时间来表示缓存的有效期。

Cache-conttrol:max-age=25920000
2-2-1、Cache-control常用的字段值

max-age：缓存的有效时间；
max-stale：表示可接受的缓存最大过期时间；
min-frash：表示客户端希望在指定的时间内获得最新的响应；
must-revalidate：如果超过了max-age的时间，在使用缓存时浏览器必须向服务器验证缓存是否可用；
no-cache：在使用缓存时，需要想服务器验证缓存是否过期；
no-store：禁止任何形式的缓存；
only-if-cached：表示浏览器只接受已经缓存过的响应；
public：表明响应可以被任何对象（包括：发送请求的客户端，代理服务器，等等）缓存。
private：表明响应只能被客户端缓存；

2-2-2、Cache-control的设置规则：

此外，在HTTP/1.1之前，如果想设置no-cache，需要通过Pragma字段来设置：

Pragma:no-cache

Cache-control 的优先级高于Expires,为了兼容HTTP/1.0和HTTP/1.1，实际项目中两个字段都会设置。

三、协商缓存

当强缓存未命中时，就需要使用协商缓存，由服务器来判断是否要使用缓存。

浏览器会先去请求缓存数据库，从而获得一个标示。再将这个标示发送给服务器，通过服务器来判断缓存是否可以使用，如果可以使用服务器就会返回304；如果缓存不能使用，服务器就会返回200和新的缓存，浏览器在收到新缓存后会将它们放入缓存服务器中，供下次使用。

协商缓存有两组标示：Last-Modified && If-Modified-Since和Etag && If-None-Match。

3-1、Last-Modified && If-Modified—Since

服务器发送响应时会带有Last-Modified头字段，用来表示文件最后一次修改的时间。
当浏览器请求资源时，如果缓存过期，就会向服务器发送请求，并将响应中Last-Modified字段的值附加在If-Modified—Since字段上。
服务器通过通过对比修改时间来判断是否过期，如果没有过期就会返回304，如果过期就返回200和新的资源。

但是这种方式有一些缺点：
如果资源属于周期性写入，但每次并没有写入新的内容，这样就无法充分的使用缓存资源。

3-2、Etag和If-None-Match

Etag是通过对资源的标示，在服务器对资源的响应中将这个标示放入Etag字段中，这样浏览器在请求时就将这个标示放入到If-None-Match中发送给服务器，服务器同样进行比较，命中返回 304, 不命中返回新资源和 200。

Etag 的优先级高于 Last-Modified

四、按照缓存的位置分类

缓存按照位置也可以分为：memory cache和disk cache。
memory cache属于浏览器的自身行为，一般对于派生资源例如：脚本，图片，css样式等会缓存在memory cache中，memory cache的缓存行为不受响应头的影响，同时当标签页被关闭后，缓存中的资源也会被清除。

disk cache是从磁盘中获取缓存资源，等待下次访问时不需要重新下载资源，而直接从磁盘中获取，强制缓存和协商缓存都属于这种类型。

memory cache的优先级要大雨disk cache，当浏览器读取缓存时会先去memory cache中寻找。

五、CDN与缓存

当服务器使用来CDN来进行缓存时，dns服务器解析会最终将域名的解析权交给CNAME指向的CDN专用DNS服务器，DNS服务器会讲负载均衡的设备的IP发回给浏览器，所以浏览器的请求会经由负载均衡设备发往合适的CDN服务器。
CDN服务器也会通过响应字段来维护自己的缓存数据，当用户向CDN服务器发送请求时，如果缓存没有过期就会直接将缓存发回给浏览器。如果缓存过期那么CDN服务器也会从上级服务器拉取最新的数据进行缓存并返回给客户端。
